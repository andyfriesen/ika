ika Python Reference

Constants

List of constants found in the ika module.

Blend Modes

A list of constants signifying which blending mode to use in drawing functions.

ika.AddBlend -> int == 3 (read)

Each pixel's RGBA values will be added to the values of the pixel behind it.

ika.AlphaBlend -> int == 2 (read)

All alpha values will be treated normally. This is the default blend mode.

ika.Matte -> int == 1 (read)

All nonzero alpha pixels will be plotted at full opacity. Pixels with 0 alpha will not be plotted. This blend mode only works with Canvases.

ika.Opaque -> int == 0 (read)

All alpha values will be treated as if they were 255.

ika.SubtractBlend -> int == 4 (read)

Each pixel's RGBA values will be subtracted from the values of the pixel behind it.

Miscellaneous

Other constants.

ika.Version -> string (read)

Returns the version of ika currently being used (as a string), including any alpha/beta suffixes.

Functions

Global functions in the ika module.

ika.Delay(int duration)

Freezes the engine for duration 'ticks'. (one tick is 1/100th of a second). The engine will simply stop doing anything until the given time has elapsed. If you wish to have the engine process normally for a time, use Wait instead.

ika.EntitiesAt(int x, int y, int width, int height, int layer) -> list entList

Returns entList, a list containing all entities within the rectangle (x, y, x+width, y+height) on the given layer. If there are no entities there, an empty list is returned.

ika.Exit([string message])

Exits ika immediately, displaying message in a small window containing an OK button, if specified.

ika.GetCameraTarget() -> ika.Entity target

Returns the entity that the camera is following (target), or None if it is free. See also: ika.SetCameraTarget.

ika.GetCaption() -> string caption

Returns the caption currently being displayed in the ika window title bar. See also: ika.SetCaption.

ika.GetFrameRate() -> int frameRate

Returns the current engine frame rate, frameRate, in frames per second. (Note: There is no function to automatically set frame rate. You must write your own method for this or use a library. A tutorial on this subject is in the works.)

ika.GetPlayer() -> ika.Entity player or None

Returns the current player entity, or None if there isn't one. See also: ika.SetPlayer.

ika.GetRGB(int color) -> tuple components (int red, int green, int blue, int alpha)

Returns a 4-tuple components containing the red, blue, green, and alpha values of the color passed, respectively. See also: ika.RGB.

ika.GetTime() -> int ticks

Returns the number of ticks elapsed since the engine started execution.

ika.HookRetrace(function hook)

Adds the function hook to the retrace queue. (It will be called whenever the map is drawn, whether by Map.Render or by other means.) See also: ika.UnhookRetrace, ika.HookTimer, and ika.UnhookTimer.

ika.HookTimer(function hook)

Adds the function hook to the timer queue. The function will be called 100 times per second. (This feature should be used sparingly, as it will cause serious problems if the queue cannot be executed in less than 1/100th of a second.) See also: ika.UnhookTimer, ika.HookRetrace, and ika.UnhookRetrace.

ika.Log(string message)

Writes message to ika.log, if logging is enabled in user.cfg. The same effect can be achieved by using the Python print command, as it is overloaded.

ika.ProcessEntities()

Performs a tick (1/100th of a second) of entity AI. Calling this 100 times a second will cause entities to move around as if the engine was in control.

ika.Random(int min, int max) -> int value

Returns a random integer value greater than or equal to min and less than max. (ie. min <= value < max)

ika.Render([list layerList [int layerIndex, ...]])

Draws the map and entities. layerList is a sequence of integers; ika will only draw these layers (in the order given). If layerList is omitted, all layers will be drawn, in their predefined order (set in the editor). If layerList is omitted, the screen is cleared to black before rendering begins.

ika.RGB(int red, int green, int blue[, int alpha]) -> int color

Creates a 32-bit color value from the four color levels passed. If alpha is omitted, it is assumed to be 255 (opaque). See also: ika.GetRGB.

ika.SetCameraTarget(ika.Entity target)

Sets the camera target to the entity specified (target). If None is passed instead, the camera remains stationary, and can be altered with the ika.Map.xwin and ika.Map.ywin properties. See also: ika.GetCameraTarget.

ika.SetCaption(string caption)

Sets the caption currently being displayed in the ika window title bar. See also: ika.GetCaption.

ika.SetMapPath(string mapPath)

Sets the folder for ika to load both map and tileset files from. Make sure to end the string with a slash ("/").

ika.SetPlayer(ika.Entity player)

Sets the player entity to the entity passed. The player entity is the entity that moves according to user input. Passing None instead unsets any player entity that may have been previously set. See also: ika.GetPlayer.

ika.UnhookRetrace([function hook])

Removes the function hook from the retrace queue if it is present. If not, the call does nothing. If hook is omitted, then the list is cleared in its entirety. See also: ika.HookRetrace, ika.HookTimer, and ika.UnhookTimer.

ika.UnhookTimer([function hook])

Removes hook from the timer queue if it is present. If not, the call does nothing. If hook is omitted, then the timer queue is cleared in its entirety. See also: ika.HookTimer, ika.HookRetrace, and ika.UnhookRetrace.

ika.Wait(int ticks)

Runs the engine for a number of ticks, disallowing player input. Unlike Delay, Wait causes entities to be processed, the tileset to be animated, and the map drawn.

Objects

Global objects in the ika module. Each one represents a type of resource used by ika. Some are singletons that represent various parts of ika's core functionality.

Input

A singleton which is an interface for hardware input devices such as the keyboard and mouse. More information on utilizing the keyboard, joystick, and mouse can be found here.

Control Objects

A Control object is an object representing some key, button, or analog input device. These objects are not inherently exposed to you as the scripter, but the various Input Device objects (keyboard, mouse, and joystick) return these for your use. Control objects have these properties: control.Position() -> int value -- int representing whether or not a control is currently pressed down. control.Pressed() -> int value -- int representing whether or not a control has been pressed since the last time control.Pressed was called. The value becomes 0 after control.Pressed is called and will not change to 1 unless the control is pressed again. control.Delta() -> int value -- int representing amount control has changed since last calling it. For keyboard, this is the same as control.Pressed. control.onpress -> function -- function that is called when the control is pressed. Defaults to None; setting this is not required. control.onunpress -> function -- function that is called when the control is released. Defaults to None; setting this is not required. See also: ika.Input.keyboard, ika.Input.mouse, and ika.Input.joysticks.

ika.Input.cancel -> Control (read/write)

Gets the standard "Cancel" control. Has no inherent effect. Defaults to Escape (ika.Input.keyboard["ESCAPE"]). See also: ika.Input.enter.

ika.Input.down -> Control (read/write)

Gets the standard "Down" control. Moves the player down when the engine has control. Defaults to the down arrow key (ika.Input.keyboard["DOWN"]). See also: ika.Input.up, ika.Input.left, and ika.Input.right.

ika.Input.enter -> Control (read/write)

Gets the standard "Enter" control. Activates entities and zones when the engine has control. Defaults to Enter/Return (ika.Input.keyboard["RETURN"]). See also: ika.Input.cancel.

ika.Input.keyboard -> Input Device (read)

Gets the keyboard device. This device contains all of the Controls on the keyboard, which can be accessed as if Input.keyboard were a dictionary. Use ika.Input.keyboard[keyname] to get the desired Control. ika.Input.keyboard[string keyName] -> Control -- method for retrieving keyboard Control objects. Click here for information on using Control objects. Click here for the list of available keycodes. Click here for a more in-depth explanation.

ika.Input.joysticks -> tuple ([Input Device, ...]) (read)

Gets a tuple containing all the connected joystick devices. Joystick support is limited, but functional. Joystick features are accessed via several member variables of a joystick object. ika.Input.joysticks[int index] -> Joystick -- method for retrieving Joystick objects. The variables below are members of Joystick objects which each contain that joystick's relevant Control objects. joystick.axes -> list -- list containing the value of each axis. joystick.reverseAxes -> list -- list containing the reversed value of each axis. joystick.buttons -> list -- list containing the position of each button. (Since joysticks come in different varieties, the length of these lists may differ.) Click here for information on using Control objects. Click here for a more in-depth explanation.

ika.Input.left -> Control (read/write)

Gets the standard "Left" control. Moves the player left when the engine has control. Defaults to the left arrow key (ika.Input.keyboard["LEFT"]). See also: ika.Input.up, ika.Input.down, and ika.Input.right.

ika.Input.mouse -> Input Device (read)

Gets the mouse device. This device contains all of the Controls on the mouse (listed below), which are accessed as normal member variables of ika.Input.mouse. ika.Input.mouse.control -> Control -- method for retrieving mouse Control objects. mouse.x -> Control -- control holding x-axis information. This Control's Position function returns a value from 0 to the value of ika.Video.xres, or the screen's horizontal resolution. mouse.y -> Control -- control holding y-axis information. This Control's Position function returns a value from 0 to the value of ika.Video.yres, or the screen's vertical resolution. mouse.left -> Control -- control holding left mouse button information. mouse.middle -> Control -- control holding middle mouse button information. mouse.right -> Control -- control holding right mouse button information. (On mice with no middle button, middle button functionality is emulated by having both left and right buttons pressed simultaneously. ika.Input.mouse.middle will reflect this.) Click here for information on using Control objects. Click here for a more in-depth explanation.

ika.Input.right -> Control (read/write)

Gets the standard "Right" control. Moves the player right when the engine has control. Defaults to the right arrow key (ika.Input.keyboard["RIGHT"]). See also: ika.Input.up, ika.Input.down, and ika.Input.left.

ika.Input.Unpress()

Unsets the Pressed property of all controls. Has no effect on their positions. Individual controls can be unpressed by simply calling their Pressed method, discarding the result.

ika.Input.up -> Control (read/write)

Gets the standard "Up" control. Moves the player up when the engine has control. Defaults to the up arrow key (ika.Input.keyboard["UP"]). See also: ika.Input.down, ika.Input.left, and ika.Input.right.

ika.Input.Update()

Updates the state of any attached input devices. Also gives the OS a chance to do background tasks. Continuous loops should call this occasionally to give the OS time to perform its tasks. (Note: If this function is not called in a loop, the player will have no control in that loop. This also means they will not be able to close the program with Alt-F4. You should ALWAYS use this in a loop unless you have a very specific reason not to do so.)

Map

A singleton which represents the current map. Since this is a singleton, it is always the map the engine is currently using; there is no way to create a different object for a different map.

ika.Map.entities -> dict {string entName: ika.Entity ent, ...} (read/write)

Gets a dictionary of entities currently tied to the map. Entities not in this list will not be affected by ika.Map.ProcessEntities. See also: ika.Map.GetAllEntities.

ika.Map.FindLayerByName(string layerName) -> int layerIndex

Returns the layerIndex of the first layer with the given layerName, or None if no such layer exists.

ika.Map.GetAllEntities() -> list entityList [ika.Entity ent, ...]

Creates a list of every single existing entity, and returns it. This includes entities created through code. See also: ika.Map.entities.

ika.Map.GetLayerName(int layerIndex) -> string

Returns the name of the layer with the given index.

ika.Map.GetLayerPosition(int layerIndex) -> tuple position (int x, int y)

Returns a tuple containing the position on the map, in pixels, of the layer with index layerIndex. This position corresponds the top-left corner of the layer. See also: ika.Map.SetLayerPosition.

ika.Map.GetLayerProperties(int layerIndex) -> tuple layerProperties (string label, int width, int height, int wrapx, int wrapy)

Returns a tuple layerProperties containing information about the map layer specified. This function is subject to an out-of-bounds error if the layerIndex passed is not valid.

ika.Map.GetLayerTint(int layerIndex) -> int tintColor

Returns the tint color of the layer with the given index layerIndex. Color values are created by ika.RGB. See also: ika.Map.SetLayerTint.

ika.Map.GetMetaData() -> dict metadata {string key: string value, ...}

Creates a dictionary containing the map's metadata, and returns it.

ika.Map.GetObs(int x, int y, int layerIndex) -> int obs

Returns 1 if the tile at (x, y) is obstructed, or 0 if not.

ika.Map.GetParallax(int layerIndex) -> (int xmul, int xdiv, int ymul, int ydiv)

Returns a 4-tuple in the format (x multiplier, x divisor, y multiplier, y divisor) containing parallax settings for the layer layerIndex specified. See also: ika.Map.SetParallax.

ika.Map.GetTile(int x, int y, int layerIndex) -> int tileIndex

Returns the index of the tile at (x, y) on the layer layerIndex specified.

ika.Map.GetWaypoints() -> list waypoints [tuple waypoint (string name, int x, int y), ...]

Returns a list of tuples in the format of (name, x, y), one for each waypoint defined within the editor. (Note: Waypoints are currently unimplemented and so, do nothing. This function is useless. You won't use it.)

ika.Map.GetZones(int layerIndex) -> list zones [tuple zoneInfo (int x, int y, int width, int height, function activateScript), ...]

Returns a list of tuples (zones) containing information about every zone on the layer layerIndex specified. The tuples are in the format (x, y, width, height, activateScript). activateScript is a function that executes when the zone is entered or activated adjacently.

ika.Map.height -> int (read)

Gets the height of the current map, in pixels. This value determines only how far the camera may scroll and is independent from individual layer heights. See also: ika.Map.width.

ika.Map.layercount -> int (read)

Gets the number of layers on the current map.

ika.Map.module

Gets the Python module corresponding to the current map.

ika.Map.numtiles -> int (read)

Gets the number of tiles in the current tileset.

ika.Map.SetLayerName(int layerIndex, string newName)

Sets the name of the layer with layerIndex to newName. Subsequent GetLayerName and FindLayerByName will use this as the layer's name, as if it were set in the editor. (Obviously, this value is only temporary and will not change in the map file itself.)

ika.Map.SetLayerPosition(int layerIndex, int x, int y)

Sets the layer's position to the (x, y) coordinates, in pixels, specified. This position corresponds the top-left corner of the layer. See also: ika.Map.GetLayerPosition.

ika.Map.SetLayerTint(int layerIndex, int tintColor)

Sets the tint color of the layer with index layerIndex to tintColor. All tiles on the layer, when drawn, will be tinted this color. Color values are created by ika.RGB. See also: ika.Map.GetLayerTint.

ika.Map.SetObs(int x, int y, int layerIndex, int obs)

If obs is nonzero, the tile at (x, y), in tiles, is obstructed, else it is unobstructed. See also: ika.Map.GetObs.

ika.Map.SetParallax(int layerIndex, int xmul, int xdiv, int ymul, int ydiv)

Sets the specified layer's parallax settings according to the multipliers and divisors given. If either of the divisors are zero, a parallax value of 0/1 will be used for that axis. See also: ika.Map.GetParallax.

ika.Map.SetTile(int x, int y, int layerIndex, int tileIndex)

Sets the tile at (x, y) of layer layerIndex to the tile with index tileIndex in the current tileset. These indeces can be determined by loading the tileset in the map editor.

ika.Map.Switch(string mapFilename)

Switches the current map to the map file specified. The new map's AutoExec event is executed, if it exists.

ika.Map.tilesetname -> string (read/write)

Gets or sets the filename of the current tileset. This will change the tiles used in the map instantly but will not change the value in the map file itself.

ika.Map.tileheight -> int

Gets the height of tiles, in pixels, in the current tileset. See also: ika.Map.tilewidth.

ika.Map.tilewidth -> int (read)

Gets the width of tiles, in pixels, in the current tileset. See also: ika.Map.tileheight.

ika.Map.title -> string (read/write)

Gets or sets the map's title. This parameter has no effect and is merely metadata displayed in the map editor.

ika.Map.width -> int (read)

Gets the width of the current map, in pixels. This value determines only how far the camera may scroll and is independent from individual layer widths. See also: ika.Map.height.

ika.Map.xwin -> int (read/write)

Gets or sets the x coordinate, in pixels, of the camera. This position corresponds to the top-left corner of the screen. The value cannot exceed the map width. See also: ika.Map.ywin and ika.Map.width.

ika.Map.ywin -> int (read/write)

Gets or sets the y coordinate, in pixels, of the camera. This position corresponds to the top-left corner of the screen. The value cannot exceed the map height. See also: ika.Map.xwin and ika.Map.height.

Video

Interface for ika's graphics engine. This object is a singleton, which means there can only be one instance of it at any time.

ika.Video.Blit(ika.Image image, int x, int y[, int blendMode])

Draws the image at (x, y) at its original size. See also: Blend Modes.

ika.Video.ClearScreen()

Clears the screen by drawing a black rectangle over it. The effect, like with all other drawing functions, will not appear until ika.Video.ShowPage is called.

ika.Video.ClipBlit(ika.Image image, int x, int y, int clipx, int clipy, int clipw, int cliph[, int blendMode])

Draws a portion of the image starting at the coordinates (clipx, clipy) with a width of clipw and height of cliph at screen coordinates (x, y). See also: Blend Modes.

ika.Video.ClipScreen([int left = 0, int top = 0, int right = ika.Video.xres, int bottom = ika.Video.yres])

Clips the video display to the rectangle specfied. All drawing operations will be confined to this region; any pixels drawn outside of this range will be omitted. Calling ClipScreen with no arguments will reset the clipping rect to its default setting. (the whole screen).

ika.Video.GetClipRect()

Return a 4-tuple containing the currently set clip rect. (as set by Video.ClipScreen)

ika.Video.DistortBlit(ika.Image image, tuple upleft (int upleftX, int upleftY), tuple upright (int uprightX, int uprightY), tuple downright (int downrightX, int downrightY), tuple downleft (int downleftX, int downleftY)[, int blendmode])

Draws the image onscreen, stretched to the four points specified. Remember that the point tuples go in clockwise order around the image, starting at the top left. See also: Blend Modes and ika.Video.TintDistortBlit.

ika.Video.DrawEllipse(int cx, int cy, int rx, int ry, int color[, int filled, int blendMode])

Draws an ellipse, centered at (cx, cy), of radius rx and ry on the X and Y axis, respectively. If filled is non-zero, the ellipse is filled in; else, it is drawn as an outline (including if the parameter is omitted). Color values are created by ika.RGB. See also: Blend Modes.

ika.Video.DrawLine(int x1, int y1, int x2, int y2, int color[, int blendMode])

Draws a straight line from (x1, y1) to (x2, y2) in the color specified. Color values are created by ika.RGB. See also: Blend Modes.

ika.Video.DrawPixel(int x, int y, int color[, int blendMode])

Draws a pixel at (x, y) with the color specified. Color values are created by ika.RGB. See also: Blend Modes.

ika.Video.DrawRect(int x1, int y1, int x2, int y2, int color[, int filled, int blendMode])

Draws a rectangle with (x1, y1) and (x2, y2) as the top-left and bottom-right corners, respectively. If filled is omitted or zero, an outline is drawn; else, it is filled in. See also: Blend Modes.

ika.Video.DrawTriangle(tuple point1 (int x1, int y1, int color1), tuple point2 (int x2, int y2, int color2), tuple point3 (int x3, int y3, int color3))

Draws a triangle onscreen. Each point is the specific color specified, and the other pixels in the triangle are blended between based on their positions. Color values are created by ika.RGB.

ika.Video.GrabCanvas(int x1, int y1, int x2, int y2) -> ika.Canvas screenGrab

Grabs a rectangle from the screen, copies it to a canvas (screenGrab), and returns it. See also: ika.Image.

ika.Video.GrabImage(int x1, int y1, int x2, int y2) -> ika.Image screenGrab

Grabs a rectangle from the screen, copies it to an image (screenGrab), and returns it. See also: ika.Image.

ika.Video.ScaleBlit(ika.Image image, int x, int y, int width, int height[, int blendMode])

Draws the image at (x, y), stretching it out to the width and height given. See also: Blend Modes.

ika.Video.ShowPage()

Flips the back and front video buffers. This must be called after you have called all desired drawing functions, or the scene will never be presented to the player. Calling this function is the only way to update the screen's contents. This function is not guaranteed to preserve the contents of the screen, so it is advised to redraw the entire screen, instead of incrementally drawing.

ika.Video.TileBlit(ika.Image image, int x, int y, int width, int height[, float scalex[, float scaley[, int blendMode]]])

Draws the image onscreen, "tiling" it as necessary to fit the rectangle specified. scalex and scaley are floating point values used as a scale factor. The default is 1 for each. See also: Blend Modes and ika.Video.TintTileBlit.

ika.Video.TintBlit(ika.Image image, int x, int y, int tintColor[, int blendMode])

Draws the image onscreen, using tintColor to 'tint' the image. Each pixel is multiplied by tintColor. The resultant values are then scaled before the pixel is plotted. In English, this means that RGBA(255, 255, 255, 255) is a normal blit, while RGBA(0, 0, 0, 255) will leave the alpha channel intact, but reduce all pixels to black. (effectively drawing a silhouette). See also: Blend Modes, ika.Video.TintDistortBlit, and ika.Video.TintTileBlit.

ika.Video.TintDistortBlit(ika.Image image, tuple upleft (int upleftX, int upleftY, int upleftTintColor), tuple upright (int uprightX, int uprightY, int uprightTintColor), tuple downright (int downrightX, int downrightY, int downrightTintColor), tuple downleft (int downleftX, int downleftY, int downleftTintColor)[, int blendMode])

Combines the effects of ika.Video.DistortBlit and TintBlit. Each corner can be tinted individually. The corners, if not the same, are smoothly interpolated across the image. Remember that the point tuples go in clockwise order around the image, starting at the top left. Color values are created by ika.RGB. See also: Blend Modes.

ika.Video.TintTileBlit(ika.Image image, int x, int y, int width, int height, int tintColor[, float scalex[, float scaley[, int blendMode]]])

Draws the image onscreen, "tiling" it as necessary to fit the rectangle specified. Also tints each pixel the color tintColor. scalex and scaley are floating point values used as a scale factor. The default is 1 for each. See also: Blend Modes, ika.Video.TintBlit, and ika.Video.TileBlit.

ika.Video.xres -> int (read)

Gets the horizontal resolution of the current display mode, in pixels. The default, if left unchanged in user.cfg, is 640. See also: ika.Video.yres.

ika.Video.yres -> int (read)

Gets the vertical resolution of the current display mode, in pixels. The default, if left unchanged in user.cfg, is 480. See also: ika.Video.xres.

Canvas

A software representation of an image that can be manipulated easily. Canvases cannot be blit onto the screen directly; they must first be transformed into images. See also: ika.Image.

ika.Canvas(string imageFilename) -> ika.Canvas canvas

Loads the image specified by 'imageFilename' into a new canvas.

ika.Canvas(int width, int height) -> ika.Canvas canvas

Creates a new, blank canvas of the specified size (width, height) in pixels.

ika.Canvas.Blit(ika.Canvas destCanvas, int x, int y, int blendMode)

Draws the canvas on destCanvas, at position (x, y) See also: Blend Modes.

ika.Canvas.Clear([int color])

Sets every pixel on the canvas to the colour given. If color is omitted, black is used.

ika.Canvas.Clip([int x, int y, int width, int height])

Sets the clipping rectangle of the canvas. Blitting to the canvas will be confined to the clip rectangle; blitting the canvas on others will only blit the clipped region. If no arguments are specified, the clip rectangle is reset to cover the whole canvas.

ika.Canvas.DrawText(ika.Font font, int x, int y, string text)

Draws the string text starting at (x,y) with the font specified. No word wrapping is done.

ika.Canvas.Flip()

Flips the contents of the canvas on the X axis (turns it upside down). See also: ika.Canvas.Mirror.

ika.Canvas.GetPixel(int x, int y) -> int color

Returns the pixel at position (x, y) on the canvas, as a packed 32bpp RGBA color. Color values are created by ika.RGB. See also: Blend Modes.

ika.Canvas.height -> int (read)

Gets the height, in pixels, of the canvas. See also: ika.Canvas.width.

ika.Canvas.Mirror()

Mirrors the contents of the canvas along the Y axis (left to right). See also: ika.Canvas.Flip.

ika.Canvas.Resize(int width, int height)

Resizes the canvas to the size specified. No scaling takes place; if the dimensions given are smaller than the existing image, it is cropped. Blank, transparent space is added when the canvas is enlarged.

ika.Canvas.Rotate()

Rotates the contents of the canvas 90 degrees, clockwise. The size of the canvas does not change, so if the canvas was not square, any pixels outside the range will be cropped.

ika.Canvas.Save(string filename)

Writes the image to the filename specified in PNG format ("myimage.png", for example).

ika.Canvas.ScaleBlit(ika.Canvas destcanvas, int x, int y, int width, int height[, int blendmode)

Draws the image on destCanvas, at position (x, y), scaled to (width, height) pixels in size. See also: Blend Modes.

ika.Canvas.SetPixel(int x, int y, int color)

Sets the pixel at (x, y) on the canvas to the color specified. This function totally disregards alpha. It *sets* the pixel, in the truest sense of the word. Color values are created by ika.RGB.

ika.Canvas.TileBlit(ika.Canvas destCanvas, int x, int y, int width, int height[, int offsetx, int offsety, int blendMode])

Tiles the canvas within the region specified, on destCanvas. The tiling is offsetn(moved) by offsetx and offsety (both default to zero).

ika.Canvas.width -> int (read)

Gets the width, in pixels, of the canvas. See also: ika.Canvas.height.

ika.Canvas.WrapBlit(ika.Canvas destCanvas, int x, int y, int width, int height, [int offsetx, int offsety, int blendMode])

Tiles the image within the region specified, on destcanvas. The tiling is offset (moved) by offsetx and offsety. See also: Blend Modes.

Music

A hunk of music data. Music can be in .mod, .s3m, .xm, .it, .mp3, or .ogg format. This type is intended for music and not sound effects. It has more definable attributes than Sound objects. Multiple music files can be loaded at once. See also: Sound.

ika.Music(string musicFilename) -> ika.Music music

Loads a music file into memory and stores it in an ika.Music object. The music engine which ika uses, audiere, can sometimes be picky about certain music file specifications. If one setting causes problems (such as a crash or an error), try re-encoding the file at a different bitrate. This problem is intermittent and fairly rare.

ika.Music.loop -> int (read/write)

Gets or sets the looping status of the music. If non-zero, the music file will loop when it encounters a loop point (if it is a .mod, .s3m, .xm, or .it file) or when the song finishes. If zero, the song will stop once finished and will need to be played again to start.

ika.Music.pan -> float (read/write)

Gets or sets the panning (horizontal position) of the music. 0.0 is far left; 1.0 is center; 2.0 is far right. The default is 1.0.

ika.Music.Pause()

Pauses the music's playback. Has no effect if the music had previously finished playing and is not looping. See also: ika.Music.Play.

ika.Music.pitchshift -> float (read/write)

Gets or sets the music's pitch shift setting. This setting determines the frequency at which the music is played. The relationship between the value of this parameter and its effect is uncertain at this time. The default is 1.0. (Note: This parameter may not have an effect with music objects. Testing is being done.)

ika.Music.Play()

Plays the music. The music will start from the beginning if it had previously finished playing and is not looping; otherwise, it will start where paused. See also: ika.Music.Pause.

ika.Music.position -> int (read/write)

Gets or sets the position in milliseconds of the music. The range is from 0 to the length in milliseconds of the song. Setting it past these bounds will cap it at the nearest endpoint. (Note: This value is not precise, as setting it to 0 will not completely reset the song. Testing is being done. This is an Audiere issue.)

ika.Music.volume -> float (read/write)

Gets or sets the volume of the music. 0.0 is silence and 1.0 is 100% volume. The default is 1.0.

Sound

A hunk of sound data, like a sound effect. Sounds can be in .wav, .mp3, or .ogg format. This type is not intended for music. Multiple sound files can be loaded at once. See also: Music.

ika.Sound(string soundFilename) -> ika.Sound sound

Loads a sound file into memory and stores it in an ika.Sound object. The sound engine which ika uses, audiere, can sometimes be picky about certain sound file specifications. If one setting causes problems (such as a crash or an error), try re-encoding the file at a different bitrate. This problem is intermittent and fairly rare.

ika.Sound.pan -> float (read/write)

Gets or sets the panning (horizontal position) of the sound. 0.0 is far left; 1.0 is center; 2.0 is far right. The default is 1.0.

ika.Sound.Pause()

Pauses the sound's playback. Has no effect if the sound had previously finished playing. See also: ika.Sound.Play.

ika.Sound.pitchshift -> float (read/write)

Gets or sets the sound's pitch shift setting. This setting determines the frequency at which the sound is played. The relationship between the value of this parameter and its effect is uncertain at this time. The default is 1.0.

ika.Sound.Play()

Plays the sound. The sound will start from the beginning if it had previously finished playing; otherwise, it will start where paused. See also: ika.Sound.Pause.

ika.Sound.volume -> float (read/write)

Gets or sets the volume of the sound. 0.0 is silence and 1.0 is 100% volume. The default is 1.0.

Image

A hardware-dependant image object. Objects of this type can be blit (drawn) onto the screen. Image objects are immutable, so drawing them onto the screen is all that can be done.

ika.Image.Blit(int x, int y[, int transparent])

Draws the image on the screen at (x, y). If transparent is specified and 0 the image is opaque. Otherwise, alpha blending is enabled.

ika.Image.DistortBlit(tuple upleft (int upleftX, int upleftY), tuple upright (int uprightX, int uprightY), tuple downright (int downrightX, int downrightY), tuple downleft (int downleftX, int downleftY)[, int transparency])

Blits the image scaled to the four points specified. Remember that the points go clockwise around the image in argument order.

ika.Image.height -> int (read)

Gets the height, in pixels, of the image. See also: ika.Image.width.

ika.Image.ScaleBlit(int x, int y, int width, int height[, int transparent])

Blits the image, but stretches it out to the dimensions specified in (width, height). If transparent is non-zero or omitted, alpha blitting is enabled.

ika.Image.width -> int (read)

Gets the width, in pixels, of the image. See also: ika.Image.height.

Entity

An object representing an interactive object on a map.

ika.Entity(int x, int y, int layerIndex, string spriteFilename) -> ika.Entity ent

Creates a new entity at pixel coordinates (x,y) on the layerIndex specified. The new entity uses the sprite indicated.

ika.Entity.actscript -> function (read/write)

Gets or sets the function called when the entity is activated.

ika.Entity.adjacentactivate -> function (read/write)

Gets or sets the function called when the entity touches the player.

ika.Entity.curframe -> int (read)

Gets the entity's currently displayed frame. If ika.Entity.specframe is non-zero, curframe will return the value of specframe. Otherwise, it will return the frame of the current animation currently being displayed.

ika.Entity.DetectCollision() -> ika.Entity touchingEnt

If an entity is touching the entity, then it is returned. None is returned if there is no entity touching it. See also: ika.Entity.Touches.

ika.Entity.direction -> int (read/write)

Gets or sets the entity's direction.

ika.Entity.Draw([int x, int y])

Draws the entity at the position specified. x and y default to the position where the entity would normally be drawn, given the window position and the entity position. See also: ika.Entity.Render.

ika.Entity.entobs -> int (read/write)

This property determines whether or not this entity is obstructed by other entities. If nonzero, the entity is unable to walk through entities whose isobs property is set. See also: ika.Entity.isobs and ika.Entity.mapobs.

ika.Entity.GetAllAnimScripts() -> dict scripts {string scriptName: string script, ...}

Returns a dict containing all animation scripts defined in the entity's current spriteset. See also: ika.Entity.GetAnimScript.

ika.Entity.GetAnimScript(string scriptName) -> string script

Returns the animation script with the name given from the entity's current spriteset. An empty string is returned if no such script exists. See also: ika.Entity.GetAllAnimScripts.

ika.Entity.hotheight -> int (read)

Gets the height of the entity's hotspot. (An entity's hotspot is the area of the sprite that is obstructed.) See also: ika.Entity.hotwidth.

ika.Entity.hotwidth -> int (read)

Gets the width of the entity's hotspot. (An entity's hotspot is the area of the sprite that is obstructed.) See also: ika.Entity.hotheight.

ika.Entity.hotx -> int (read)

Gets the X position of the entity's hotspot. (An entity's hotspot is the area of the sprite that is obstructed.) See also: ika.Entity.hoty.

ika.Entity.hoty -> int (read)

Gets the Y position of the entity's hotspot. (An entity's hotspot is the area of the sprite that is obstructed.) See also: ika.Entity.hotx.

ika.Entity.IsMoving() -> int moving

If the entity is moving, the result is 1. If not, it is 0.

ika.Entity.isobs -> int (read/write)

This parameter determines whether or not the entity obstructs other entities. If nonzero, the entity prevents other entities whose entobs property is set from walking through it. See also: ika.Entity.entobs and ika.Entity.mapobs.

ika.Entity.layer -> int (read/write)

Gets or sets the index of the layer that the entity exists on.

ika.Entity.mapobs -> int (read/write)

If non-zero, the entity is unable to pass through obstructed tiles on its current layer. See also: ika.Entity.entobs and ika.Entity.isobs.

ika.Entity.movescript -> string (read/write)

Gets or sets the entity's current move script. A tutorial on movescript formatting can be found (here).

ika.Entity.MoveTo(int x, int y)

Directs the entity to move towards the position specified. (Note: This is not an instant teleport; the entity will continue to move toward this position until it is stopped or ika.ProcessEntities is not called.) See also: ika.Entity.Stop.

ika.Entity.name -> string (read/write)

Gets or sets the entity's name. This value has no inherent effect and is more or less for your own convenience only.

ika.Entity.Render([int x, int y])

Draws the entity at the position specified. Calling this will call a ika.Entity.renderscript if one is set. For this reason, DO NOT CALL THIS IN A RENDER SCRIPT, or you will enter an infinite loop. See also: ika.Entity.Draw.

ika.Entity.renderscript -> function (read/write)

Gets or sets the function called when the entity is rendered. Do not call ika.Entity.Render in this script, or you will enter an infinite loop.

ika.Entity.specanim -> string (read/write)

Gets or sets the specific animation to use instead of normal animations. If 0, normal animations will be used.

ika.Entity.specframe -> int (read/write)

Gets or sets the specific frame to use instead of normal animations. If 0, normal animations will be used. See also: ika.Entity.curframe.

ika.Entity.speed -> int (read/write)

Gets or sets the entity's speed, in pixels/second. Defaults to 100.

ika.Entity.sprite -> string (read/write)

Gets or sets the sprite used to display the entity. This parameter is just a string fileanem that tells ika where the .ika-sprite file is. The entity's appearance will change immediately after changing this value.

ika.Entity.Stop()

Directs the entity to stop whatever it is doing. See also: ika.Entity.MoveTo.

ika.Entity.Touches(ika.Entity ent) -> int touches

Returns true if the entity is touching entity ent, else False. See also: ika.Entity.DetectCollision.

ika.Entity.Update()

Performs 1/100th of a second of entity AI. Calling this 100 times a second will cause the entity to move around as if the engine was in control. If you do not wish to call ika.ProcessEntities, call this function for every entity you wish to update.

ika.Entity.visible -> int (read/write)

Gets or sets whether or not the entity will be drawn when in view.

ika.Entity.Wait(int ticks)

Causes the entity to halt for the given interval before resuming motion.

ika.Entity.x -> int (read/write)

Gets or sets the entity's X position on its current layer (in pixels). See also: ika.Entity.y.

ika.Entity.y -> int (read/write)

Gets or sets the entity's Y position on its current layer (in pixels). See also: ika.Entity.x.

Font

A proportional bitmap font. Variable-width fonts are supported. There are several tools to create .fnt files compatible with ika.

ika.Font(string fontFilename) -> ika.Font font

Creates an object of type ika.Font and loads the font into memory. fontFilename is a string that contains the filename to a font file (with extension .fnt).

ika.Font.CenterPrint(int x, int y, string text)

Prints a string of text on screen, with x as the center point rather than the leftmost point. This means that the character in the middle of the string will be centered on this point; it also means that half of the string's width will be to the left and half will be to the right. See also: ika.Font.Print and ika.Font.RightPrint.

ika.Font.height -> int (read)

Gets the height, in pixels, of the font. All glyphs in a font are the same height.

ika.Font.letterspacing -> int (read/write)

Gets or sets the letter spacing, in pixels, of the font. This value determines how much padding is added between letters when printed. The default is 0.

ika.Font.Print(int x, int y, string text)

Prints a string of text on screen at (x, y). This print is left-justified. See also: ika.Font.CenterPrint and ika.Font.RightPrint.

ika.Font.RightPrint(int x, int y, string text)

Prints a string of text on screen, with x as the rightmost point rather than the leftmost point. This means that the last letter's right edge will fall on the coordinate x specified. See also: ika.Font.Print and ika.Font.CenterPrint.

ika.Font.StringWidth(string text) -> int width

Returns how many pixels in width the passed string would be when printed in the given font. This function includes the value of the current font's ika.Font.letterspacing in its calculation.

ika.Font.tabsize -> int (read/write)

Gets or sets the tab size, in pixels, of the font. To insert a tab into text, include a "t" in the string.

ika.Font.width -> int (read)

Gets the width, in pixels, of the widest glyph in the font. Note that fonts in ika are variable-width, not monospace, so not all glyphs have to be the same width.
